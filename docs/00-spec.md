# เกมลูกเต๋า — Design Spec (v0.1)

> เอกสารฉบับนี้คือ "รัฐธรรมนูญ" ของเกม ทุกการตัดสินใจด้านดีไซน์ต้องไม่ขัดกับข้อตกลงในนี้
> ตัวเลขทั้งหมดเป็นค่าเริ่มต้นสำหรับ playtest — ปรับได้ แต่โครงสร้างห้ามแตก

---

## 1. แก่นของเกม (Core Identity)

- เกม **PvE เสี่ยงดวง** สไตล์ poker-dice ผสม RPG สวมใส่อุปกรณ์
- กราฟิกแนว **วาดมือบนกระดาษสมุด** (ดินสอ, จินตนาการเด็ก)
- 3 อาชีพเริ่มต้น: **Swordman / Thief / Mage**
- จุดขาย: **costume น่ารัก + สัตว์เลี้ยงช่วยตี**

---

## 2. รัฐธรรมนูญข้อ 1 — โมเดลรายได้ (ห้ามละเมิด)

| ขายด้วยเงินจริง | หาจากการเล่นเท่านั้น |
|---|---|
| Costume / skin | อุปกรณ์ที่ weight ลูกเต๋า |
| สีลูกเต๋า (cosmetic) | ความแรงสกิล / จำนวนลูกเต๋า |
| Pet skin / effect สวย | การ์ด buff จากมอนสเตอร์ |

**กฎเหล็ก:** ของที่กระทบ "พลัง" = เล่นเอาเท่านั้น / ของที่ขายเงินจริง = "หน้าตา" เท่านั้น
→ รักษาคำว่า "แฟร์ ไม่ pay-to-win" ไว้ตลอด
→ **ยังไม่มี gacha ใน MVP** ทำ core combat ให้สนุกก่อน

---

## 3. กลไกลูกเต๋า

- เริ่มเกมมี **1 ลูก** → สูงสุด **5 ลูก** (ปลดล็อกตามความก้าวหน้า)
- ลูกเต๋า 1 ลูก = 6 หน้า แต่ละหน้ามี **เลข (1–6)** + **สี (6 สี)**
- **สีสุ่มใหม่ทุกครั้งที่ทอย** — เลขกับสีเป็นอิสระต่อกัน
- poker hand ใช้ **เลขอย่างเดียว** / สีเป็น **ตัวคูณ** แยกแกนกัน

---

## 4. Damage Formula (ระบบบวก — เลขกลม)

```
ดาเมจ = ผลรวมแต้ม + โบนัส poker + โบนัสสี + โบนัสอุปกรณ์/ลูกน้อง
```

> **หลักการ:** ทุกอย่างเป็น "บวก" ไม่มีคูณทศนิยม → เลขกลมเสมอ คุมไม่ให้ดาเมจบานง่าย
> **2 แกน:** เลข (poker) + สี (โบนัส) รวมเป็น "ความแรงของกอง" แล้วเอาไปลงช่องเดียว → ตีครั้งเดียวต่อเทิร์น

**โบนัส poker hand** (คิดเฉพาะลูกเต๋าในช่องที่ลง)

| Hand | เงื่อนไข | โบนัส |
|---|---|---|
| เลขโดด | ไม่เข้า hand | +0 |
| Pair | คู่ | +5 |
| Two Pair | สองคู่ | +10 |
| Three of a Kind | สามตัว | +15 |
| **Straight** | **เรียง 3 ลูกขึ้นไป** | +25 |
| Full House | สาม+คู่ | +30 |
| Four / Five Kind | สี่–ห้าตัว | +40 / +60 |

**โบนัสสี** (สีเดียวกันในกอง — ขั้นบันได เลี้ยงความฟินตลอดเกม)

| สีเดียวกัน | โบนัส |
|---|---|
| 2 ลูก | +5 |
| 3 ลูก | +10 |
| 4 ลูก | +20 |
| 5 ลูก (full flush) | +40 — jackpot |

> **สีทำหน้าที่เดียว = โบนัสความแรง** ไม่เกี่ยวกับจำนวนครั้งตี / ไม่เกี่ยวกับเวทย์ (กันสับสน)

> *ตัวอย่าง:* ช่องโจมตีลง 3 ลูก ได้ `4,4,4` (Three of a Kind, รวมแต้ม 12) สีแดง 3 ลูก
> → 12 + 15 (three kind) + 10 (สี 3 ลูก) = **37 ดาเมจ**

---

## 5. Core Loop — 1 เทิร์น (โมเดล Allocate)

1. **เปิดเทิร์น** — โชว์ HP เรา / HP มอนสเตอร์ / **intent ของมอนสเตอร์** (จะตีแรงเท่าไหร่)
2. **ทอย** — เต๋าทั้งหมดกลิ้งออกมา โชว์เลข+สี
3. **จัดสรร** — ลากเต๋าแต่ละลูกลงช่อง `[⚔️ โจมตี] [🛡️ ป้องกัน] [✨ สกิล]`
   - ระบบโชว์ผลแบบ real-time ขณะลาก (poker hand + ดาเมจ อัปเดตสด)
   - poker คิด**เฉพาะลูกในช่องนั้น** → เกิด tension "ทุ่มตีให้ได้ straight หรือแบ่งไปกัน"
4. **ยืนยัน** — คำนวณ → แสดงผล → มอนสเตอร์ทำ intent → จบเทิร์น

---

## 6. ระบบรอบข้าง

- **การ์ดมอนสเตอร์ดรอป** = passive buff สวมในช่องอุปกรณ์ (เช่น "สกิลสายฟ้า +20%")
  - ⚠️ คุม scope: เริ่มจากการ์ดจำนวนน้อย ทำ template buff ซ้ำได้ อย่าออกแบบ logic เฉพาะทุกใบ
- **เศรษฐกิจในเกม:** มอนสเตอร์ดรอปเงิน → ซื้อยา/ของ/ขายของ
  - ⚠️ ระวัง "ใครยาเยอะคนนั้นชนะ" — ต้องออกแบบ sink/source ให้ดวงยังตึง
- **สัตว์เลี้ยง → "ลูกน้อง":** คือมอนสเตอร์ที่เจอระหว่างทาง จับมาเลี้ยงได้ (reuse art+logic ตัวเดียวสองหน้าที่)
  - **Passive:** อยู่ในทีมก็มีผล เช่น เพิ่ม weight โอกาสออกสีแดง +5%
  - **Active:** เรียกออกมาแล้วให้ "ลูกเต๋าพิเศษ" ที่มีคุณสมบัติเฉพาะ (ตามทาง A — pet = ลูกเต๋าเพิ่ม)
  - ⚠️ **รัฐธรรมนูญ:** ขายได้แค่ skin/สีของลูกน้อง — ห้ามขายตัวลูกน้องที่ช่วยตี หรือโอกาสจับ

---

## 7. เทคโนโลยี

- **Engine: Godot** (2D, ฟรีจริง, workflow วาดมือดี, ไม่มีดราม่า license, ทำคนเดียวไหว)
- เป้าหมาย: iOS + Android
- **MVP แรก: combat loop + 1 อาชีพ + มอนสเตอร์ 2–3 ตัว** ยังไม่ต้องมี gacha/server/payment

---

## ❓ จุดที่ยังไม่ล็อก (ค้างไว้คุยต่อ)

- อุปกรณ์ weight ทำงานยังไงเชิงตัวเลข (เพิ่มโอกาสเลข? โอกาสสี? — เหมือน passive ลูกน้องไหม)
- ความแตกต่างเชิง mechanic ของ 3 อาชีพ (ไม่ใช่แค่ตัวเลข) ← **ผมเชียร์ให้เคลียร์ก่อน กระทบ MVP**
- ระบบความก้าวหน้า: ปลดล็อกลูกเต๋าลูกที่ 2–5 ด้วยอะไร
- ช่อง "สกิล" ทำงานยังไง — ลูกเต๋าที่ลงช่องสกิล แปลงเป็นอะไร (เงื่อนไขปล่อยสกิล?)
