## การตัดสินใจตั้งต้น (จากการปรึกษาก่อนเริ่มโปรเจกต์)

**โมเดลรายได้ → cosmetic-only, ไม่มี gacha ใน MVP**
เหตุผล: รักษาคำว่า "แฟร์ ไม่ pay-to-win" + เกมอินดี้คนเดียวสู้ power-gacha กับเจ้าใหญ่ไม่ได้ + gacha ต้องมี server/payment/กฎหมาย ซึ่งหนักเกินจะเริ่มด้วย

**การตัดสินใจในเกม → โมเดล Allocate (ทอยทีเดียว แล้วลากลงช่อง โจมตี/กัน/สกิล)**
เหตุผล: ตรงกับที่เจ้าของอธิบายแต่แรก (มีเต๋าโจมตี/กัน/เวทย์) + ลื่นบนมือถือกว่า re-roll

**poker → คิดเฉพาะลูกในช่องที่ลง + straight ลดเหลือ 3 ลูกเรียง**
เหตุผล: allocate ทำให้แต่ละช่องเหลือลูกน้อย ถ้า straight ต้อง 5 ลูกจะแทบเกิดไม่ได้ + สร้าง tension "ทุ่มตีหรือแบ่งกัน"

**สีลูกเต๋า → 6 สี (1 สี/1 เลข) สุ่มสีใหม่ทุกครั้งที่ทอย**
เหตุผล: เจ้าของเลือก 6 สี เพื่อความสวยและขายเป็น cosmetic ได้ / สุ่มสีใหม่ทำให้เลขกับสีเป็นอิสระ ทาง A ทำงานสะอาด

**สี → ทำหน้าที่เดียว = โบนัสความแรง (ไม่ใช่จำนวนครั้งตี/ไม่ใช่เวทย์)**
เหตุผล: เคยคิดเอาสีไปผูกจำนวนครั้งตี แต่มันชนระบบ allocate และทำให้ลูกเต๋า 1 ลูกมี 2 หน้าที่พร้อมกัน = สับสน

**Formula → ระบบบวก (ไม่ใช่คูณ)**
เหตุผล: เจ้าของอยากได้เลขกลม ปัญหาเลขไม่กลมมาจากการคูณทศนิยมซ้อน เปลี่ยนเป็นบวกแก้ที่ราก + คุมดาเมจไม่ให้บานง่ายกว่า

**ตีครั้งเดียวต่อเทิร์น**
เหตุผล: กองลูกเต๋าในช่องโจมตีรวมเป็นก้อนเดียว สะอาด เข้ากับ allocate

**สัตว์เลี้ยง → "ลูกน้อง" = มอนสเตอร์ที่จับมาเลี้ยง (ทาง A: pet = ลูกเต๋าเพิ่ม)**
เหตุผล: reuse art+logic ลด scope มหาศาล + pet ในรูปลูกเต๋าพิเศษไม่กลบ core combat / passive+active

**Engine → Godot 4**
เหตุผล: 2D ฟรีจริง workflow วาดมือดี ไม่มีดราม่า license ทำคนเดียวไหว

**โครงสร้าง → CLAUDE.md + game-designer subagent 1 ตัว (ยังไม่สร้าง agent อื่น)**
เหตุผล: subagent คือภาระดูแล ไม่ใช่ความปลอดภัย / agent ที่ความรู้ว่างเปล่าช่วยอะไรไม่ได้ / สร้างเพิ่มเมื่อมีของจริงให้จำ

## 🅿️ Parking Lot (ไอเดียค้าง — ห้ามทำก่อน gate Phase 0 ผ่าน)

**[2026-06-20] โครงสร้างโลก: เดินเมืองต่อเมืองแบบ RO — ✅ เจ้าของยืนยัน "ไม่เอา open world"**
- สถานะ: ทิศทางล็อกแล้ว (โซน/เดินเมือง) แต่ **รายละเอียดยัง PARKED** จนผ่าน gate + เริ่ม Phase 1 (maps.md)
- รูปแบบ: แผนที่แยกโซน เดินเมือง→ทุ่ง→ดันเจียน ผ่าน portal — ไม่ใช่โลกไร้รอยต่อ
- เหตุผล: RO เองก็เป็นแบบโซน / open world จริง = กับดักอินดี้คนเดียว / โซนทำคนเดียวบนมือถือไหว

**[2026-06-20] ระบบตีบวก/อัปเกรดอุปกรณ์ (PARKED → Phase 1)**
- ไอเดียเจ้าของ: ใส่ของได้ตั้งแต่ต้น · ล่ามอนได้วัตถุดิบ+เงิน → เอามาตีบวกเสริมพลัง (เช่น ของเพิ่มเลือด) · **มีโอกาสไม่สำเร็จ แต่ "ของไม่แตก"** (ต่างจาก RO ที่แตกได้)
- จุดประสงค์: สร้าง loop "ล่าของ → พัฒนาตัวเอง" ให้ผู้เล่นอยากเล่นต่อ
- ⚠️ ต้องคุม scope: ตรงรัฐธรรมนูญ (ของกระทบพลัง = เล่นเอา) / เงิน+วัตถุดิบต้องออกแบบ sink/source ไม่ให้ "ของเยอะชนะ"
- เงื่อนไขปลดล็อก: Phase 1

**[2026-06-20] ระบบลูกน้อง (capture) — คำถามค้าง: ทำยังไงให้ผู้เล่น "อยากจับ" (PARKED → Phase 1)**
- spec มีโครงแล้ว (จับมอน = ลูกน้อง passive/active, pet = ลูกเต๋าพิเศษ) แต่ "hook ที่ทำให้อยากจับ" ยังไม่ออกแบบ
- ต้องตอบตอน Phase 1 ตอนทำมอน + ระบบจับจริง

**[2026-06-20] Habitat มอนสเตอร์ (PARKED → Phase 1) — เหตุผลที่เจ้าของอยากเปิดแมพ**
- ไอเดีย: มอนแต่ละชนิดอยู่ตาม "ถิ่นอาศัย" (ป่า/ถ้ำ/ทะเลทราย/หนองน้ำ/ภูเขาไฟ) แต่ละโซนมี monster pool + ช่วงเลเวล + บอสของตัวเอง
- ⚠️ สำคัญ: habitat ทำได้ด้วย "โซน" แบบ RO (แต่ละแมพมี spawn list ของตัวเอง) — **ไม่ต้อง open world** (open world ยังเป็นกับดักอินดี้คนเดียว)
- ผูกกับ hook ลูกน้อง: ลูกน้องบางตัวจับได้เฉพาะในถิ่นของมัน → เป็นเหตุผลให้ผู้เล่นออกเดินทาง+สำรวจ (ตอบคำถาม "ทำไงให้อยากจับ")
- เงื่อนไขปลดล็อก: Phase 1

## Phase 1 — implement + บาลานซ์ (2026-06-20)
**สกิลแทน Heal → tier เลขกลม (ข้อ3 ok) รวมเป็นดาเมจก้อนเดียว (ข้อ5 ok)** — Novice 💢 ทุ่มพลัง [4,9,18] · Heal = ของ Acolyte (Phase 2)
**บาลานซ์ → เปลี่ยนจาก pwr-scaling เป็น level-based** — เพราะ pwr-scaling ทำให้ของ/สกิล/ลูกน้อง "ไม่จำเป็น" (ชนะ 100% ใน 3 เทิร์น) · level-based ทำให้บอสเป็นกำแพงต้องฟาร์ม (ของจำเป็นจริง)
**เริ่ม 1 เต๋า (กลับจาก 2)** — เพราะมีสกิลแล้ว 1 เต๋าเล่นได้จริง + ล้มบอสไหว
**ลูกน้อง → เลือก active ได้ 1 ตัว** (passive เฉพาะตัวเลือก) — "เลือกว่าใช้ตัวไหน"
**เครื่องมือ: simulator `prototype/sim_play.js`** — บอตเล่นด้วยสูตรจริง ใช้จูนบาลานซ์ (ต้อง sync BAL กับเกมเสมอ) · ผล: เล่นจบ 100%, ตาย ~0.5/รอบ, บอส Novice เปล่า ชนะ 4% (ของจำเป็น)
**4 โซน + บอสมังกร = จบ Phase 1** (ป่า→ทุ่ง→ถ้ำ→ภูเขาไฟ) · gate คนจริงยังไม่ผ่าน

## Phase 1 v3 — เพิ่มความลึก (2026-06-20)
**เลเวลอัพ** (XP/kill, +5HP+1atk/lv, xpNeed 4+lv×3) — แก้ "ตีแล้วล่องลอย" · ลดเลือดมอนต้นเกมเหลือ 20-30
**Speed** = ตีก่อน(initiative) + ลุ้นทอยซ้ำ — เร็วกว่ามอน=ฆ่าได้ไม่โดนสวน+ลุ้น 🔄, ช้ากว่า=โดนสวนแม้ฆ่าได้ · กางเกง(legs)ให้ spd
**Elite tier** ทุกโซน (โผล่ ~12%, จับได้ passive แรง, ดรอป net) — ระหว่าง General/Boss
**การ์ดเป็น socket ในอุปกรณ์** — เกราะ(หมวก/เสื้อ/กางเกง) 1 ช่อง, ดาบ/โล่ 1-4 ช่องสุ่มตาม tier ("หลังๆช่องเยอะ") · การ์ดแยกหมวด weapon/shield/armor · ของเป็น instance (มี sk[])
**ไอเทมจับ** 🪤 ตาข่าย (+30% จับ, ซื้อ/ดรอป elite) + เหยื่อ (likes)
**ร้านค้า** ซื้อยา/ตาข่าย + ขายของ(แตะ) · **เพิ่ม drop rate** (ของ 60%, การ์ด 22%, elite ดรอปหนัก)
**ช่อง Feedback ในเกม** → POST /feedback → เขียน `feedback.txt` ผ่าน _serve.py ( tester ส่งผ่านลิงก์ได้เลย, แนบสถานะเกมอัตโนมัติ)
**sim v3 (3000 รอบ):** จบเกม 100% · ตาย ~0.21/รอบ · ไม่ตายเลย 86% · เลเวลจบ ~11

## Phase 1 v4 — รีบาลานซ์ให้ท้าทาย + UX (2026-06-20)
**ของบวกน้อยลง โดยเฉพาะ def** (โล่ 1/2/4/6 แทน 2/5/9/14, สเต็ปนุ่ม) — เพราะ def เยอะทำให้มอนหมดความโหด
**ผู้เล่นโตช้าลง:** HP 30 + **3/lv** (จาก 5), **atk/def +1 สลับทุก 2 เลเวล** (จาก atk +1 ทุกเลเวล) — เกมท้าทายขึ้น
**Elite/Boss โหดขึ้น:** eliteHP×1.45 int×1.3 spd7 · bossHP×2.0 int×1.55 spd6
**บอส: มีการ์ด+วิจัย+จับได้** (โอกาสต่ำ 8%+ตาข่าย/เหยื่อ, cap 50%) — จับบอสได้ลูกน้อง passive แรง (ent +50HP, ram +12atk, mino +14def, dragon +18atk+แดง) · จับบอส = นับเป็นล้ม (ปลดเต๋า/ชนะ)
**UX:** ดรอปโชว์รายการ+จำนวน+EXP ชัดใน log/toast · เลเวลอัพแจ้งว่าได้อะไร (❤️+3, สเตตที่สลับ)
**sim v4:** จบเกม 100% · ตาย ~0.59/รอบ · ไม่ตายเลย 72% · บอสป่า Novice 2% (ต้องฟาร์ม) Elite 95%
